Plant Health Identifier

Overview

I built this tool to help small-scale growers and plant lovers easily identify possible plant diseases just by using pictures. The idea was to make something simple and clean that anyone could use—no technical experience needed. The app lets users upload images through an easy-to-use interface. Behind the scenes, it’s set up with a solid backend structure that supports image handling and diagnostic features. The goal is to make plant care smarter and more accessible, even for people growing from home.

Tools & Technologies

  • HTML5 & Bootstrap: clean layout and responsive structure
  • JavaScript: form handling, file preview, and event control
  • Prepared API integration: design accommodates external ML or REST APIs

Features

  • Custom file upload: user-friendly UI styled with Bootstrap and custom CSS
  • Image preview: allows users to verify their photo before submission
  • API-ready design: supports asynchronous request handling

Challenges & Solutions

One of the biggest challenges was making sure the file upload logic was clean, precise, and bug-free. Even small mistakes—like a missing function call or an incorrect file path—could break the whole feature or cause silent errors. To overcome this, I took extra care with testing and debugging. I broke the code into smaller pieces, double-checked each part, and used clear console messages to trace behavior. This helped me make sure everything ran smoothly and did exactly what it was supposed to do.

Impact & Future Plans

This project helped me grow as a frontend developer and sparked my interest in computer vision, especially for plant care and agriculture. It showed me how powerful image-based tools can be in helping everyday users solve real problems. Building this app made me think more about how technology can support sustainability and empower small growers with tools that are usually out of reach.
